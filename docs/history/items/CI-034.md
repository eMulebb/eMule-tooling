## Summary

The release package command must make package provenance unambiguous. It should
record the selected `main` source accurately and reject dirty source inputs that
would make app/build/test/tooling provenance unclear.

## Acceptance Criteria

- [x] `package-release` records the app source variant, branch, and commit used
      for the package.
- [x] `package-release` rejects dirty relevant source/doc inputs before writing
      release assets.
- [x] Package manifests identify build and tooling commits used by the package.
- [x] Packaging tests or focused command evidence cover the dirty-input guard.
