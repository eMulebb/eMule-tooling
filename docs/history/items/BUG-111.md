## Summary

Release-facing app URLs must point at the policy-owned `emulebb` namespace.
The beta must not direct update checks, release links, or help links to
superseded `itlezy/eMule` locations.

## Acceptance Criteria

- [x] Release update UI opens an `emulebb` release URL.
- [x] Release update API requests use an `emulebb` GitHub API namespace.
- [x] Online help uses the current policy-owned documentation location.
- [x] Focused release-update tests cover the selected namespace.
- [x] A source grep confirms no stale public release/update/help URL remains in
      active app code.
