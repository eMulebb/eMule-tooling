## Summary

Prove that native REST, qBittorrent-compatible, and Torznab-compatible surfaces
have not drifted from their intended contracts. Native `/api/v1` behavior must
remain clean and typed; adapter-specific quirks must stay isolated in the qBit
and Torznab compatibility layers.

## Acceptance Criteria

- [ ] OpenAPI/native REST route manifest matches current implementation.
- [ ] Safe native REST routes pass the contract smoke suite.
- [ ] Destructive native REST routes have explicit confirmation or intent
      inventory coverage.
- [ ] Native REST malformed input returns typed JSON errors where expected.
- [ ] qBittorrent-compatible auth rejects missing, wrong, and stale sessions.
- [ ] qBittorrent-compatible malformed booleans and hashes are rejected or
      handled according to the adapter contract.
- [ ] Torznab API key, category, query, and feed behavior is replayed.
- [ ] Adapter error shapes remain adapter-local and do not force native
      `/api/v1` response aliases or envelopes.
