## Summary

Refresh the full controller integration proof for R-1.0.1. The release claim is
that eMule BB has functional parity with Prowlarr, Radarr, Sonarr, and
aMuTorrent in the supported controller lanes.

## Acceptance Criteria

- [ ] aMuTorrent browser smoke passes against current `main` or records an
      accepted public-network inconclusive result.
- [ ] Prowlarr add/test/search passes with redacted live terms and API keys.
- [ ] Radarr and Sonarr sync the Prowlarr indexer and keep RSS, automatic
      search, and interactive search enabled.
- [ ] Radarr and Sonarr qBittorrent-compatible download-client validation
      succeeds against eMule BB.
- [ ] qBit-compatible transfer add, info, properties, files, category mutation,
      pause/resume, and delete are exercised.
- [ ] Controller reports expose live-network policy evidence:
      `BindInterface=hide.me`, empty P2P `BindAddr`, and P2P UPnP enabled.
- [ ] Nested controller/browser HTTP errors fail the suite instead of being
      reported as successful parent workflow steps.
