## Summary

Replay the highest-volume local data path: shared files. Broadband changed
startup sequencing, startup cache, known-file indexing, recursive share sync,
long-path behavior, share-ignore policy, and shared-file REST projection.

## Acceptance Criteria

- [ ] 50k shared-file stress reaches the main window and converges UI rows.
- [ ] REST shared-file rows match UI row counts before and after churn.
- [ ] Startup cache survives cached relaunch and invalidation scenarios.
- [ ] Recursive watcher churn adds, removes, and renames files without stale UI
      rows or lost REST rows.
- [ ] Long-path share, delete, shell, and config paths remain functional.
- [ ] `shareignore.dat` policy is applied additively and does not hide
      unexpected stock files.
- [ ] Stock profile migration does not lose known-file metadata.

## Validation

- `pwsh -File EMULE_WORKSPACE_ROOT\repos\eMule-build\workspace.ps1 live-e2e -Config Release -Platform x64 -LiveSuite shared-files-ui`
- Existing focused tests for long-path, known-file, shared-file, and startup
  cache behavior through supported `workspace.ps1` commands.
