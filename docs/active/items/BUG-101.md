## Summary

The CI-018 live smoke still fails before tree churn can run when the Shared Files
profile contains a 10k-node recursive shared-directory tree. The hardened
fixture now limits file rows to `1024` and keeps `11329` observable nodes, so
the remaining blocker is startup/list population under a large recursive shared
directory set rather than bulk file hashing.

## Evidence

- Test harness commit `8d63a45` reshaped the tree stress fixture to `1024`
  branch files plus `10305` directories, selects the stress subtree before the
  initial row-count gate, and gives the heavy scenario a `180s` main-window
  wait.
- Live artifact
  `repos\eMule-build-tests\reports\shared-files-ui-e2e\20260508-125931-eMule-main-release\tree-refresh-stress-10k\result.json`
  failed with `Timed out waiting for eMule main window. Last value: None`.
- The artifact reports `observable_node_count=11329`,
  `expected_row_count=1024`, `directory_count=10305`, and
  `shared_directory_count=10305`.
- A root-only experiment reached the main window but scanned only `2`
  directories and `0` files, so it is not a valid recursive tree stress gate.

## Execution Plan

1. Reproduce with the hardened CI-018 fixture and capture whether startup is
   hung parsing `shareddir.dat`, building the Shared Files tree, or blocked
   before startup profiling initializes.
2. Add a startup trace or narrow diagnostic before and after shared-directory
   list load so the next artifact identifies the exact phase.
3. Audit shared-directory startup parsing for quadratic string/path operations
   and unbounded UI-thread work when thousands of directories are configured.
4. Preserve legacy recursive sharing semantics; do not replace recursive
   directory coverage with root-only sharing because that hides the failing
   surface.
5. Fix the earliest confirmed bottleneck with minimal product drift.
6. Rerun the CI-018 live smoke with `tree-refresh-stress-10k` and record row
   convergence, churn completion, and resource deltas.

## Acceptance Criteria

- The hardened `tree-refresh-stress-10k` profile reaches the main window within
  the documented live-test timeout.
- The Shared Files list and REST `/api/v1/shared-files` converge to `1024`
  rows before and after churn.
- The fixture keeps at least `10000` observable tree/list nodes.
- Resource deltas after churn are recorded and reviewed for R1 thresholds.
- CI-018 is updated with passing smoke artifact evidence.
