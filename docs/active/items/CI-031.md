## Summary

Prove that R-1.0.1 packages are buildable, complete, and consistent with the
workspace release naming policy. The app diff includes project, manifest,
resource, language, site-config, data-file, and packaging-sensitive changes.

## Acceptance Criteria

- [ ] x64 Release app build passes through `workspace.ps1`.
- [ ] ARM64 Release app build passes through `workspace.ps1`.
- [ ] Release package rehearsal produces `eMule-broadband-1.0.1-x64.zip` and
      `eMule-broadband-1.0.1-arm64.zip` naming.
- [ ] Packages include required binaries, manifests, templates, Geo/IPFilter
      support data, flags, language resources, license/readme material, and
      controller-facing docs.
- [ ] Packages exclude retired or unsupported artifacts such as stale `Win32`
      outputs.
- [ ] Version metadata, update-check metadata, and release tag naming agree on
      `emule-bb-v1.0.1`.
- [ ] Fresh unpack smoke can launch the app with a clean profile.

## Validation

- `pwsh -File EMULE_WORKSPACE_ROOT\repos\eMule-build\workspace.ps1 build-app -Config Release -Platform x64`
- `pwsh -File EMULE_WORKSPACE_ROOT\repos\eMule-build\workspace.ps1 build-app -Config Release -Platform ARM64`
- Supported package/release rehearsal commands through
  `EMULE_WORKSPACE_ROOT\repos\eMule-build\workspace.ps1`.
