## Summary

Smoke the visible stock workflows most likely to regress after Broadband UI,
preference, resource, language, list-control, tray, and keyboard changes.

## Acceptance Criteria

- [ ] Main window opens, reaches ready state, and exposes expected core panes.
- [ ] Preferences open, save, reload, and preserve stock plus Broadband keys.
- [ ] Pro-user controls remain visible without breaking stock defaults.
- [ ] Keyboard shortcuts and mnemonic changes do not steal text-entry or
      dialog navigation behavior.
- [ ] Tray icon, balloon mode, restore/minimize, and meter icon drawing do not
      leak GDI objects or lose state.
- [ ] Representative language/resource DLLs build and load current command IDs.
- [ ] List controls touched by stale-row hardening still sort, refresh, and
      display backing rows.

## Validation

- `pwsh -File EMULE_WORKSPACE_ROOT\repos\eMule-build\workspace.ps1 build-app -Config Release -Platform x64`
- UI-focused live smoke through supported `workspace.ps1` commands.
- Representative language/resource build or load smoke through supported
  workspace commands.
