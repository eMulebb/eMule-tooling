## Summary

Prove that download state, part-file state, direct-download state, and metadata
persistence remain durable after Broadband changes. This gate focuses on user
data safety and restart behavior.

## Acceptance Criteria

- [x] Part-file save, backup, promotion, and restart probes pass.
- [x] `known.met`, `known2.met`, `server.met`, cancelled-file metadata, and Kad
      preference snapshots survive failure and restart probes.
- [x] Direct-download cancellation, timeout, close-time failure, and owner
      shutdown behavior are replayed.
- [x] Download completion hook runs only when expected and reports failures.
- [x] Pause, resume, delete, rename, and majority-name display do not corrupt
      download rows or persistence.
- [x] Community-compatible paths remain unchanged where Broadband features are
      disabled.
