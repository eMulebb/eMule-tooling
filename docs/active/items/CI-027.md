## Summary

Prove that download state, part-file state, direct-download state, and metadata
persistence remain durable after Broadband changes. This gate focuses on user
data safety and restart behavior.

## Acceptance Criteria

- [ ] Part-file save, backup, promotion, and restart probes pass.
- [ ] `known.met`, `known2.met`, `server.met`, cancelled-file metadata, and Kad
      preference snapshots survive failure and restart probes.
- [ ] Direct-download cancellation, timeout, close-time failure, and owner
      shutdown behavior are replayed.
- [ ] Download completion hook runs only when expected and reports failures.
- [ ] Pause, resume, delete, rename, and majority-name display do not corrupt
      download rows or persistence.
- [ ] Community-compatible paths remain unchanged where Broadband features are
      disabled.

## Validation

- `pwsh -File EMULE_WORKSPACE_ROOT\repos\eMule-build\workspace.ps1 validate`
- Focused persistence and download tests through supported `workspace.ps1`
  test commands.
- Release x64 smoke for completion-hook and direct-download behavior.
