## Summary

Release-facing app URLs must point at the policy-owned `emulebb` namespace.
The beta must not direct update checks, release links, or help links to
superseded `itlezy/eMule` locations.

## Acceptance Criteria

- [ ] Release update UI opens an `emulebb` release URL.
- [ ] Release update API requests use an `emulebb` GitHub API namespace.
- [ ] Online help uses the current policy-owned documentation location.
- [ ] Focused release-update tests cover the selected namespace.
- [ ] A source grep confirms no stale public release/update/help URL remains in
      active app code.

## Validation

- `python -m emule_workspace validate`
- Focused release-update tests through the supported workspace test entrypoint.
