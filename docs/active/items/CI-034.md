## Summary

The release package command must make package provenance unambiguous. It should
record the selected `main` source accurately and reject dirty source inputs that
would make app/build/test/tooling provenance unclear.

## Acceptance Criteria

- [ ] `package-release` records the app source variant, branch, and commit used
      for the package.
- [ ] `package-release` rejects dirty relevant source/doc inputs before writing
      release assets.
- [ ] Package manifests identify build and tooling commits used by the package.
- [ ] Packaging tests or focused command evidence cover the dirty-input guard.

## Validation

- `python -m emule_workspace validate`
- Package-focused tests when available.
- x64 and ARM64 package rehearsal after the guard lands.
