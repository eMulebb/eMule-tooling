## Summary

Replay stock-compatible search, server, Kad, bootstrap, and source-exchange
behavior after Broadband changes. This area has a large diff surface and is
where accidental protocol or lifecycle drift is most likely to hide.

## Acceptance Criteria

- [ ] Server list import, save, retry, obfuscated login, and plain fallback
      behavior are replayed.
- [ ] Kad bootstrap and nodes.dat import behavior are replayed.
- [ ] Server search and Kad search start, return rows or accepted live-network
      inconclusive results, and stop cleanly.
- [ ] Closing search tabs cancels active searches and clears UI rows safely.
- [ ] Source exchange and published-state behavior remain community-compatible.
- [ ] UDP/Kad malformed packet probes do not overread, crash, or wedge state.
- [ ] Search result ceilings and category behavior remain consistent with
      documented Broadband policy.
