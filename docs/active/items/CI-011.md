## Summary

Make the broadband release live E2E lane a first-class release gate instead of
an ad hoc collection of scripts.

## Release 1.0 Classification

**Release Gate.** This owns the operator-facing first-release command and the
result artifact shape. It should aggregate the release-critical REST,
WebServer, live-wire, aMuTorrent, and Arr controller checks without making every
future UI E2E lane block `emule-bb-v1.0.0`.

## Execution Plan

Covered by the [Release 1.0 Live E2E execution plan](../plans/RELEASE-1.0-LIVE-E2E-EXECUTION-PLAN.md).

## Acceptance Criteria

- [x] `run_live_e2e_suite.py` has stable selectable suites for every release
      lane
- [x] each suite writes a result artifact with status, command line, timings,
      and failure phase
- [x] public network failures are reported as inconclusive with diagnostics
      when the app and harness behaved correctly
- [x] the default release lane uses isolated temp profiles and does not depend
      on user profile state
- [x] aMuTorrent, Prowlarr, Radarr, and Sonarr suites are included in the
      default release lane and produce redacted artifacts
- [x] release documentation points operators to one supported command
