## Summary

Add UI-driven live search and download lifecycle coverage for the broadband
release corpus.

Search terms:

- `linux`
- `ubuntu`
- `fedora`
- `freebsd`
- `debian`
- `emule`

## Acceptance Criteria

- [ ] the UI can start server and Kad searches for the release corpus
- [ ] at least one safe candidate can be added from search results when public
      network state permits it
- [ ] pause, resume, stop, and remove are covered on a sandboxed transfer
- [ ] unsafe executable candidates are rejected by the harness
- [ ] network unavailability is inconclusive with recorded server/Kad state,
      not a false product failure
