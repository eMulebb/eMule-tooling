## Summary

Expand REST robustness coverage from representative smoke tests into a release
matrix for malformed and concurrent requests.

## Release 1.0 Classification

**Release Gate.** This validates `BUG-075`, `BUG-076`, and `BUG-077` through
selectable smoke and soak budgets. Smoke mode should be suitable for the
release gate; soak mode can be longer operator validation.

## Acceptance Criteria

- [ ] malformed JSON, non-object JSON, truncated body, wrong method, missing
      auth, wrong auth, bad route ids, and unsupported content types are covered
- [ ] failures use the stable JSON error envelope
- [ ] concurrent read and safe-mutation traffic does not hang, crash, or corrupt
      legacy WebServer session state
- [ ] stress summaries include request counts, status counts, latency bounds,
      and sampled failures
- [ ] the lane can run in short smoke mode and longer operator soak mode
