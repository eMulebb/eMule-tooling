## Summary

`CZIPFile` (~530 lines, origin: Shareaza 2002-2004) is a hand-rolled ZIP
central-directory parser + deflate extractor. **minizip** (`unzip.h`) is already
present in the `eMule-zlib` dependency tree at `contrib/minizip/` and provides
the same functionality with an actively maintained codebase.

## Consumers (3 call sites)

| File | Purpose |
|------|---------|
| `srchybrid/IPFilterDlg.cpp` | IP filter import from local `.zip` file |
| `srchybrid/PPgSecurity.cpp` | IP filter download + extract from `.zip` |
| `srchybrid/OtherFunctions.cpp` | Skin package (`.zip`) installation |

## Implementation Steps

1. Add `unzip.c`, `ioapi.c`, `iowin32.c` from `eMule-zlib/contrib/minizip/`
   to `emule.vcxproj` (and to `CMakeLists.txt` once CI-001 lands).
2. Write a thin `CMiniZipReader` wrapper matching the `CZIPFile` public API
   surface (`Open`, `GetFile`, `Close`, `CZIPFile::File`) to minimise
   consumer-side diff.
3. Migrate all 3 consumer sites to use `CMiniZipReader`.
4. Delete `srchybrid/ZIPFile.cpp` and `srchybrid/ZIPFile.h` from the project.

## Verification Checklist

- [ ] Project compiles without `ZIPFile.cpp/.h`
