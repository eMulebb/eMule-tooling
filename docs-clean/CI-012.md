## Summary

Stabilize the UI-driven Shared Files dynamic-folder lifecycle scenario so it is
usable as a release regression gate.

The current scenario exists, but the live run exposed drive-tree selection
fragility. This item turns it into a dependable test that proves UI sharing
operations and REST-visible shared-file state agree.

## Acceptance Criteria

- [ ] drive-tree node discovery works reliably on the isolated profile
- [ ] UI add/remove shared directory operations are observed through
      `/api/v1/shared-directories`
- [ ] adding and deleting files under a shared root is reflected through
      `/api/v1/shared-files`
- [ ] recursive and non-recursive roots are both covered
- [ ] failure artifacts include UI observations and REST snapshots
