## Summary

Three `// FIXME LARGE FILES` markers exist in the codebase, indicating known
incomplete code paths for files that exceed 32-bit size thresholds. These paths
are silently skipped rather than handled.

## Locations

| File | Line | Note |
|------|------|------|
| `srchybrid/AICHSyncThread.cpp` | 367 | `// FIXME LARGE FILES (uncomment)` — AICH sync incomplete for large files |
| `srchybrid/MetaDataDlg.cpp` | 345 | `// FIXME LARGE FILES` — metadata dialog may show wrong size |
| `srchybrid/SearchList.cpp` | 703 | `// FIXME LARGE FILES` — search results may handle size incorrectly |

## Impact

Users downloading or sharing files larger than the applicable threshold may see:
- Incomplete AICH hash synchronisation (`AICHSyncThread.cpp`)
- Wrong or missing file size display in the metadata dialog
- Incorrect size handling in search results

All three code paths are active — the issue is not dead code, but code guarded by
a comment instead of correct logic.

## Fix

Uncomment and test the large-file path in `AICHSyncThread.cpp`. Apply equivalent
fixes in `MetaDataDlg.cpp` and `SearchList.cpp`. Add regression test coverage
for files above the 4 GB boundary.
