## Summary

Run concurrent REST and legacy HTML traffic through the embedded WebServer to
find release-blocking hangs, crashes, and shared-state regressions.

## Release 1.0 Classification

**Release Gate.** This is the live proof that the `BUG-073` synchronization
work and the REST/legacy WebServer split hold under mixed request load. The
first release needs a short smoke budget; a longer soak budget can remain an
operator option.

## Acceptance Criteria

- [ ] concurrent REST read traffic covers app, status, snapshot, transfers,
      shared files, uploads, servers, Kad, and logs
- [ ] safe REST mutations run alongside reads without corrupting app state
- [ ] legacy HTML requests run concurrently with REST requests
- [ ] no WebServer session or bad-login state corruption appears under soak
- [ ] app shutdown remains bounded after the soak
