## Summary

Current build uses `/Wall` but does not treat warnings as errors and does not
enable SDL checks or control-flow guard. Adding these flags catches real bugs
at compile time rather than at runtime.

## Current State

- `/Wall` — all warnings enabled (already set)
- `/W4` equivalent in practice (some `/Wall` warnings suppressed via `#pragma warning(disable:...)`)
- No `/WX` (warnings as errors)
- No `/sdl` (Security Development Lifecycle checks)
- No `/guard:cf` (Control Flow Guard)
- No `/analyze` (MSVC static analysis in-build)

## Recommended Additions

```cmake
# In srchybrid/CMakeLists.txt
target_compile_options(eMule PRIVATE
    /W4           # Effective warning level (subset of /Wall, more practical)
    /WX           # Treat warnings as errors
    /sdl          # Enable SDL checks (additional security warnings + runtime checks)
    /guard:cf     # Control Flow Guard (mitigates ROP chains)
    /permissive-  # Enforce conformance mode (catches non-standard MSVC extensions)
    /Zc:__cplusplus  # Report correct __cplusplus value
)

target_link_options(eMule PRIVATE
    /guard:cf     # CFG must also be set at link time
)
```

## Warning Suppression Audit

Before enabling `/WX`, audit existing `#pragma warning(disable:...)` sites and
decide which suppressions are justified vs. which hide real issues.

## Phased Approach

1. Enable `/sdl` first — typically zero new warnings on a codebase already using
   `strcpy_s` etc.
2. Enable `/guard:cf` — linker flag, no compile-time warnings.
3. Enable `/permissive-` — may expose non-standard code; fix each instance.
4. Enable `/WX` last — only after all pre-existing warnings are resolved.

## Acceptance Criteria

- [ ] `/sdl` enabled and build passes
- [ ] `/guard:cf` enabled and binary verified with `dumpbin /loadconfig`
- [ ] `/permissive-` enabled and all non-conformance fixed
- [ ] `/WX` enabled and zero suppressed-then-hidden warnings remain

## Prerequisite

CI-001 (CMake) — hardening flags are easier to manage in CMakeLists.txt
than in vcxproj property sheets.
