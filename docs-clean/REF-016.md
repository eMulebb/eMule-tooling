## Summary

Copy the four required ResizableLib source files directly into `srchybrid/ResizableLib/` and remove the submodule. No dialog source files change. The library is MIT-licensed, ~1,500 LOC, lightly maintained, and effectively frozen ‚Äî making the in-source fork the cleanest long-term approach.

## Why

- ResizableLib is a submodule (`ppescher/resizablelib`, tracked at `master`) with a workspace patch that retargets it from `v141_xp`/SDK 8.1 to `v143`/SDK 10.0 and adds x64 Unicode configs.
- The library is stable legacy code. Last formal release: `v1.5.3` (June 2020). No meaningful upstream churn expected.
- The submodule + patch + separate build project is overhead for ~1,500 lines of stable code.
- The library is MIT-licensed ‚Äî in-source inclusion is explicitly permitted.

## Scope

ResizableLib is used via inheritance in 31 dialog/window files:

| Base class | Include | Usage count |
|------------|---------|-------------|
| `CResizableDialog` | `ResizableLib/ResizableDialog.h` | 16 files |
| `CResizablePage` | `ResizableLib/ResizablePage.h` | 11 files |
| `CResizableSheet` | `ResizableLib/ResizableSheet.h` | 4 files |
| `CResizableFormView` | `ResizableLib/ResizableFormView.h` | 1 file |

All 31 files use only the base-class interface ‚Äî no deep ResizableLib-specific calls beyond `AddAnchor`/`ArrangeLayout` in `OnInitDialog` handlers.

## Steps

1. Copy the following files from the submodule into `srchybrid/ResizableLib/`:
   - `ResizableDialog.h` / `ResizableDialog.cpp`
   - `ResizablePage.h` / `ResizablePage.cpp`
   - `ResizableSheet.h` / `ResizableSheet.cpp`
   - `ResizableFormView.h` / `ResizableFormView.cpp`
   - Any shared internal headers they include (e.g., `ResizableLayout.h/.cpp`, `ResizableGrip.h/.cpp`)

2. Update include paths in the 31 dialog files from:
   ```cpp
   #include "../ResizableLib/ResizableDialog.h"
   ```
   to:
   ```cpp
   #include "ResizableLib/ResizableDialog.h"
   ```
   (if the path changes; may be a no-op if the path was already relative to `srchybrid/`)

3. Add the copied `.cpp` files to the project (vcxproj / CMakeLists.txt) ‚Äî remove the old separate ResizableLib build project reference.

4. Remove the ResizableLib submodule: `git submodule deinit`, `git rm`, cleanup from `.gitmodules`.

5. Remove the ResizableLib workspace patch and build project from `workspace.ps1` / `deps.psd1`.

## What Does NOT Change

- All 31 dialog `.h` and `.cpp` files ‚Äî no base class changes, no `AddAnchor` removal
- Resizable dialog behavior ‚Äî identical to the submodule version
- No protocol or file transfer impact
