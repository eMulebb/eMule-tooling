## Summary

Replace `UPnPImplMiniLib` (miniupnpc) with the already-present `UPnPImplWinServ` (Windows native UPnP API) as the default UPnP implementation. This removes the miniupnpc submodule, its workspace patch, and its build project entirely — with a **one-line code change** in `UPnPImplWrapper`.

## Why

- miniupnpc is a C submodule with its own workspace patch (`emule-build-v0.72a` branch) and build plumbing.
- `UPnPImplWinServ` — a Windows-native UPnP implementation using `<upnp.h>` and `iphlpapi.lib` — already exists in the codebase and covers the same port-mapping and external-IP use cases.
- The UPnP subsystem has a clean three-layer abstraction that makes the swap trivial.

## Abstraction Layer

```
UPnPImplWrapper   ← public interface used by the rest of eMule
UPnPImpl          ← abstract base class
UPnPImplMiniLib   ← miniupnpc concrete implementation  (to remove)
UPnPImplWinServ   ← Windows native concrete implementation (to keep)
```

Only `UPnPImplWrapper` selects which implementation to instantiate. The rest of eMule (~72 files that reference UPnP settings) never sees the concrete class.

## Change Required

**In `UPnPImplWrapper.cpp` (or wherever the implementation is selected at startup):**

```cpp
// Before
m_pImpl = new UPnPImplMiniLib();

// After
m_pImpl = new UPnPImplWinServ();
```

That is the only source change. Everything else is submodule/build cleanup.

## Cleanup Steps

1. Change the one line in `UPnPImplWrapper` to instantiate `UPnPImplWinServ` instead of `UPnPImplMiniLib`
2. Remove `UPnPImplMiniLib.h` and `UPnPImplMiniLib.cpp` from the project and source tree
3. Remove miniupnpc from the vcxproj / CMakeLists.txt include and lib paths
4. Remove the miniupnpc submodule: `git submodule deinit`, `git rm`, cleanup from `.gitmodules`
5. Remove miniupnpc from `workspace.ps1` / `deps.psd1` setup scripts
6. Remove miniupnpc workspace build project (`eMule-miniupnpc/`)

## Functional Impact

- UPnP port mapping continues to work via the Windows-native implementation
- No user-visible behavior change on Windows 10+ (where the native UPnP API is fully supported)
- Users on routers not reachable by `UPnPImplWinServ` may lose automatic port mapping (rare on modern routers)
