## Summary

MKV and WebM share the same EBML magic header `1A 45 DF A3`. The current
`GetMimeType()` magic-byte table returns `video/x-matroska` for both, which
is technically incorrect for WebM files.

This is an optional quality-of-life improvement — returning `video/x-matroska`
for WebM files is not harmful for the file-info dialog use case, but it
produces incorrect MIME types when the information is used elsewhere.

## Fix

Parse the EBML DocType element deeper in the header (typically within the first
64–256 bytes) to distinguish:
- `webm` → return `video/webm`
- `matroska` → return `video/x-matroska`

## Acceptance Criteria

- `.webm` files return `video/webm` from `GetMimeType()`.
- `.mkv` files return `video/x-matroska`.
- No regression on existing format types.
